## 已完成内容

### 1. 已定位启动失败根因
- 启动报错原因已确认不是业务数据问题
- 根因是清洁发布脚本提前创建了数据库表，但没有写入 SQLite `user_version`
- 结果正式包中的数据库虽然已有完整表结构，但版本号仍是 `0`
- 应用启动时会再次走建表流程，最终报：
  - `table projects already exists`

### 2. 已修复清洁发布脚本
- 已修改：
  - `tool/prepare_clean_release.py`
- 现在脚本会从 `lib/core/constants.dart` 读取当前数据库版本号
- 创建空数据库后会同步写入：
  - `PRAGMA user_version = 5`
- 这样正式包数据库结构与版本号保持一致，不会再触发重复建表

### 3. 已增强应用侧数据库建表幂等性
- 已修改：
  - `lib/services/database_service.dart`
- 全部建表语句已改为：
  - `CREATE TABLE IF NOT EXISTS`
- 即使数据库文件已预置表结构，只要再次进入初始化，也不会因为重复建表直接崩溃

### 4. 已补回归测试并重新发布
- 新增测试：
  - `test/database_service_init_test.dart`
- 覆盖场景：
  - 数据库已存在完整 schema
  - `user_version` 被人为重置为 `0`
  - 再次执行 `DatabaseService.init()` 仍应成功
- 已重新编译 Windows release 并输出：
  - `dist/v1.1.19`
