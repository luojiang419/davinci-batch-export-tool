## 已完成内容

### 1. 已修复 xmeml 外部音频 file 定义错误
- 外部音频 `clipitem` 已保留：
  - `start/end` 作为时间线位置
  - `in/out` 作为源音频裁切范围
  - `duration` 作为片段时长
- 外部音频 `<file>` 已改为：
  - `duration` 写入源文件总时长
  - 不再写成当前裁切片段时长

### 2. 已修复共享音频文件重复定义问题
- 同一源音频文件在同一份 `xmeml` 中：
  - 只保留一份完整 `<file id=\"...\">...</file>` 定义
  - 后续复用改为 `<file id=\"...\"/>`
- 这样不会再因为重复 file id 携带不同 duration 导致 Premiere 重解释源时间码

### 3. 已补显式零基准 timecode
- 外部音频 `<file>` 现在固定输出：
  - `00:00:00:00`
  - `frame=0`
  - `displayformat=NDF`
- 当前素材没有必须保留的真实源 timecode 元数据
- 因此这版统一零基准是稳定默认值

### 4. 已补导出回归测试并通过
