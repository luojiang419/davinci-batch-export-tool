# 进度快照 85 - 多段音频合板与0916字幕基准修复完成

## 版本信息
- 当前开发版本: `v1.1.20`
- `pubspec.yaml`: `1.1.20+38`
- 阶段备份: `backup/v0.43.0`
- Git 分支: `main`

## 已完成内容

### 1. 一键合板已切换为“总音频字幕主视角”
- 自动匹配主链路不再依赖单音频文件假设
- 已改为：
  - 视频局部字幕序列
  - 对齐音频总字幕时间线
  - 再从总轨回落到具体音频文件分段
- 这样可以覆盖：
  - 录音机重启导致的多段音频接续
  - 局部字幕重复导致的错一两条问题
  - 明明整体对齐却被误判越界的情况

### 2. 已支持多段音频 segment 持久化
- 新增：
  - `SyncAudioSegment`
  - `TimelineAudioSegment`
  - `sync_audio_segments` 数据表
- `sync_results` 已补充对齐诊断字段：
  - `coarse_offset_ms`
  - `final_offset_ms`
  - `offset_mad_ms`
  - `alignment_coverage`
  - `switch_count`
  - `source_clamped_reason`

### 3. 已补 generatedAggregate 总字幕兜底
- 当工程没有显式总音频字幕时：
  - 现在会按音频文件顺序自动生成总字幕时间线
- `SubtitleSourceType` 已新增：
  - `generatedAggregate`
- “优先总字幕”查询已改为：
  - 显式 aggregate 优先
  - generatedAggregate 兜底

### 4. 时间线与导出已支持多段外部音频
- `AudioAlignService` 已改为从 `sync_audio_segments` 构建时间线
- `TimelineData` 已增加 `segments`
- `ExportService` 已支持：
  - FCPXML 多个外部音频 `audio` 元素
  - xmeml 同一视频对应多个外部音频 `clipitem`
- 默认保留原始源文件引用，不预先拼接临时 wav

### 5. 已补专项测试并通过
- 新增或扩展测试覆盖：
  - per-clip 字幕生成 `generatedAggregate`
  - 无假越界的单音频对齐
  - 多段音频切分与接续
  - 多段 xmeml / fcpxml 导出
- 已通过：
  - `flutter test`

### 6. 已跑 0916 字幕真实基准
- 基准脚本已改为默认使用：
  - `0916视频字幕.srt`
  - `0916音频字幕.srt`
- 已生成：
  - `测试合板/220822_real_sync_benchmark.xml`
  - `测试合板/220822_real_sync_benchmark.fcpxml`
  - `测试合板/220822_real_sync_benchmark.csv`
  - `测试合板/220822_real_sync_benchmark.segments.txt`
- 当前 0916 基准结果：
  - `11` 条视频全部生成结果
  - `8` 条中置信度
  - `3` 条音频过短
  - 同文件过度切片问题已收口为更合理的 `1~4` 段
  - 只剩 `1` 条出现起点不足提示
